# 圆桌（通用主持内核）

把一个**任意领域**的议题，交给若干**已沉淀的分析方法论**在**独立子线程**里相互碰撞、相互校正，
再由主持人（主线程）做元认知收束，并默认沉淀讨论精华。目标是**求真**：暴露分歧的真正根源、
用事实约束推演、逐层逼近更深的问题——而不是凑出表面共识。

> **本质：探讨，不是辩论。** 辩论是先有立场、再为它找证据（求胜）；探讨是**观点本身可以摇摆、
> 一切以事实为准**（求真）——任何一席被对方论点或新事实说服而**改变立场、下调信心，是成功不是失败**。
> 多套方法论对同一议题最终**会不会趋同**，是最有价值的看点之一；**但"达成一致"不是目的**：
> 多轮后各持己见，只要把分歧根因讲清楚，同样是合法且诚实的结论。全程任何一方都
> **不得为护住某个立场而选择性使用事实**。

## 〇、设计原则（为什么这样切）

- **机制与话题解耦**：本 Skill 只承载**与话题无关**的圆桌机制；议题、阵容、材料是每场可配项。
- **席位 = 身份（谁），不是题域分工（管什么）**：每个 lens 都是完整世界观，应对**整个议题**完整发声，
  **鼓励涉入对方擅长领域**——视角边界不预设，与"立场不预设、自行推导"一脉相承。
- **嘉宾不知道自己在圆桌里**：lens Skill 只定义"我如何分析"，**角色由主持人在本场赋予**。
  知识库能力同理——它只定义"如何写库"，"材料收集者/会议纪要"是主持人赋予的角色。
- **为何用子代理而非单线程模拟**：独立上下文 = 真正的"独立思考"，避免一个线程里互相措辞污染、
  自我强化与"假吵架真附和"。每轮按需 spawn 即可，无需常驻 agent-team。

## 一、每场可配项（开场由主持人确认）

| 可配项 | 说明 | 缺省 |
|---|---|---|
| **议题** | 本场要探讨什么（任意领域） | 必填，无则询问 |
| **嘉宾阵容** | 哪些 lens 上桌（见"二、动态选角"） | 主持人按议题提议，用户拍板 |
| **材料范围** | 事实底座要多宽 | 仅覆盖议题所需，重硬证据轻泛搜 |
| **知识库沉淀** | 是否启用材料收集者 + 会议纪要 | **默认启用**（最大化复利） |

## 二、席位与角色（通用）

| 角色 | 由谁扮演 | 职责 |
|---|---|---|
| **主持人** | 主线程（你自己，不开子代理） | 解析议题、动态选角、编排子代理、综述找裂缝、画 ASCII 结构图、提下一层问题、决定库录入、存档 |
| **事实席** | 1 个子代理（`generalPurpose`，**非 readonly**，需联网） | 开场建《事实底座》**中立骨架**；按嘉宾【数据需求】集中补采并入共享底座；终局交叉核查。**不自行预判该看什么数据** |
| **嘉宾席 ×N** | N 个子代理，各加载一个 lens/up 主 `SKILL.md` | 以该方法论**完整**分析整个议题；自行推导立场，每轮回应他席并重估本方立场 |
| **库录入席** | 1 个子代理，遵循 `library-integrate` 能力 | 把达标**来源**与终局**结论**经 Obsidian CLI 沉淀进 `library/`（默认启用） |
| **临时新视角席** | 用户指令时临时 spawn | 引入用户指定的人物/视角/方法论（只给视角不给结论，立场仍自行推导） |

> **解耦红线**：嘉宾席不直接读写知识库；一切 `library/` 读写由库录入席经 Obsidian CLI
> （`vault=everything-roundtable`）完成，遵守 `AGENTS.md`。
>
> **事实席 ≠ 库录入席**：前者为本场讨论建事实底座（进存档），后者把可复用精华写进知识库（进 `library/`）。
> **采什么由嘉宾决定**：事实底座只建中立骨架；分析所需数据由各嘉宾用【数据需求】申报、事实席按需补采——
> 主持人/事实席都不替嘉宾预判"该看什么数据"。

## 三、动态选角协议（纯动态，无注册表）

主持人**每场现扫**可用嘉宾，不维护任何注册表或固定阵容：

1. **扫描嘉宾池**：读取**与本 Skill 同级的其它 Skill**（你所在 skills 目录，Claude Code 下为 `.claude/skills/`，
   Codex 下为 `.codex/skills/`）的 frontmatter `description`，**排除**能力型（`library-*`）与本 host，其余分析型 lens 即候选。
2. **按议题相关性提议阵容**：给出 2–4 席的建议名单，每席一句话说明"它能撬动议题的哪个面/哪条分歧"。
   **优先多样性**：宁可让不同领域的方法论同桌产生张力，也不要凑同质视角。
3. **请用户拍板**：用户可增、删、替换，或临时引入新视角。
4. **缺角处理**：若现有 lens 都不贴合议题，主持人据实说明，并可临时构造"临时视角席"（只给视角、不给结论）。

> 跨域同桌完全合法：例如地缘货币方法论 + 上海地产方法论同评一个"资金往哪里去"的议题。

## 四、搜索统筹（三层）

1. **中立骨架（开场，1 次）**：事实席联网产出《事实底座》的**中立骨架**——只放客观共识：事实时间线、
   被各方引用的关键数据（数值+日期+单位）、主要信源链接、各方公开表态、已知争议点。
   **只建共同地基，不替任何方法论预判"该看什么数据"。所有席共享同一份**；碰撞只发生在"解读/推演"层，不在"事实"层。
2. **嘉宾驱动补采（每轮）**：**采什么由嘉宾决定，不由主持人或事实席自定。** 各嘉宾在发言里用
   `【数据需求｜为何需要】` 块申报本方法论所需的事实/数据；主持人汇总后交**事实席集中补采**、
   统一来源把关、并入共享底座（事实层不分裂）。嘉宾若自带有据新事实，用 `【新增事实｜来源】` 回传由主持人并入。
3. **终局交叉核查（结束前，1 次）**：事实席对讨论中引用的关键数字与关键因果做交叉验证，
   逐条标注 `✔已核实 / ⚠存疑 / ✘未能证实（含原因）`；存疑项必须进入"开放问题/风险"。

> **证据硬度分级**：每条关键事实标 **硬（官方/原始记录/成交数据）/ 中（平台数据/二手口径）/ 软（传闻/营销/小道）**；
> 事实一律带来源；**单一来源的关键数据标"待二次确认"**；推演与事实分离呈现；软证据只作核查线索，不作结论。

## 五、执行流程（主持人主线程编排）

**0. 解析议题**：从用户输入提取核心议题。无议题则询问。

**1. 动态选角 + 本场配置**：按"三"提议阵容并请用户拍板；确认材料范围与是否启用知识库沉淀（缺省启用）。

**2. 建事实底座（中立骨架）**：spawn 事实席，产出《事实底座》中立骨架（只放客观共识，不预判分析口径）。主持人向用户展示精简版（带来源与硬度标注）。

**2.5 路线确认（开场前，找你拍板一次）**：主持人基于《事实底座》提出 **3–5 个候选切入点**
（每个一句话，标明它能撬动的分歧），让用户挑选/调整/补充，确定第 1 轮要打的问题。
> 「基于事实底座，我看到这几个值得切入的角度：①… ②… ③…。你想从哪个开始？也可以直接改写或另给一个。」

这是全程唯一的"开场强确认"；之后每轮控制权在 4c 菜单里（含可随时改问）。

**3. 开场（第 1 轮）= 定锚 + 各自推导立场**：主持人只做**中立定锚**——统一关键概念与口径：
> 「先统一：本议题里 [核心概念] 的准确定义、当前事实状态、双方公认/存疑的边界，以及**这件事真正要判断的对象**是什么。在此之上，请你**用自己的方法论独立推导**对本议题的立场。」

并行 spawn 各嘉宾席（携带《事实底座》中立骨架）。每席**自行得出**并声明：
- **本方立场/判断**（一句话结论）；
- **关键依据**（用本席方法论的哪几步推出来的，引用事实带来源）；
- **信心度与最大不确定来源**；
- **本方法论所需的事实/数据**（用 `【数据需求｜为何需要】` 申报）。

主持人汇总各席【数据需求】交事实席补采、并入共享底座后，再进对话循环。

> 关键：主持人**不给任何一席预设立场**。各席立场即使最终趋同或对立，都必须是各自推导的结果。

**4. 对话循环（每轮）**：
- **4a 并行发言**：把"本轮问题 + 上一轮各席发言全文 + 最新《事实底座》"发给各嘉宾席。每席**必须回应他席**
  （质疑/反驳/补充/修正），不许自说自话；并**重估本方立场**：较上轮立场/信心是否变化、因哪条事实或谁的论点而变。
  发言结尾给 `**简言之**：` 一句话；引用数据带来源。若需更多事实，用 `【数据需求｜为何需要】` 申报（主持人交事实席补采回填）；自带有据新事实用 `【新增事实｜来源】`。
- **4b 主持人综述**（主线程亲自做，体现元认知）：
  1. 提炼本轮**最深的一条裂缝**，并判定其性质——**事实分歧 / 口径定义分歧 / 方法论假设分歧**；
  2. 生成 **ASCII 结构图**（矩阵/光谱/因果环/树形——哪种最见骨用哪种；呈现"结构"而非复述内容，
     标出因果链、正负反馈、张力维度）；
  3. 提出**下一层引导问题**（从裂缝里长出来的更深问题）。
- **4c 指令菜单**：
  ```
  【主持】：(指令: 可 / 改问 / 止 / 深入此节 / 引入新视角 / 核查此节)
  ```
  - `可`：采纳主持人提的下一层问题，进入新一轮
  - `改问`：你给出/改写下一轮问题或方向，主持人照你的来
  - `止`：结束，进入终局
  - `深入此节`：不换问题，围绕当前裂缝继续深挖
  - `引入新视角`：用户指定人物/方法论 → spawn 临时新视角席就当前问题表态
  - `核查此节`：立即让事实席对本轮关键事实/数字做一次核查再继续

  > 主持人每轮把下一题作为**默认建议**给出，但永远停下等你发令——`可` 顺着走，`改问` 你改道。

**5. 材料沉淀（默认开启，可随轮进行）**：当某条**来源**达到精选标准（支撑/挑战重要判断、提供可复用数据或验证路径、
  后续很可能再引用），主持人指派库录入席以 `input_type: source` 经 `library-integrate` 整合进 `library/`；
  **以讨论中真正起作用的来源为准**（由嘉宾引用/裂缝驱动，而非收集者随手抓）。未达标的不入库（见"十"）。

**6. 终局**：用户 `止` 后，先内部跑一遍**事实核查轮**（事实席，见四·3），再按**倒金字塔**呈现给用户
（先结论、后过程；详见"九"）：

  **A. 一句话总论**：本场 bottom line（趋同点 / 仍分歧的核心 / 最该记住的一句）。
  **B. 多方对比速览表**（核心交付，必给）：

  | 席位 | 终局结论（一句话） | 信心 | 与他席最大分歧 | 立场是否移动 |
  |---|---|---|---|---|
  | 嘉宾A | … | `▮▮▮▮▯ 80%` | … | 持平/上调/下调（因…） |
  | 嘉宾B | … | `▮▮▮▯▯ 60%` | … | … |

  **C. 趋同 / 分歧判定 + 一张图**：用 ASCII **立场光谱或矩阵**呈现各席站位与距离；文字只点明——
  哪些点已**趋同**（被事实逼出，非妥协附和）、哪些点**仍分歧**及根因（事实未决 / 口径定义 / 方法论假设）。
  **未达成一致是合法且诚实的结果，绝不为"有结论"而强行收敛。**

  ——以下为**过程层（想看分析再往下）**——

  **D. 每席推理链**：各席「关键依据 → 结论」的精炼骨架（点列，非大段）。
  **E. 立场演化轨迹**：各席**首轮 vs 终局**的立场与信心，标注移动了哪些、因哪条事实/谁的论点而移。
  **F. 开放问题 / 风险**：每条**仍分歧或存疑**项后加一句 **`判定它需要：…`**（什么证据/数据/事件能证伪或定论），
  含核查标注（✔/⚠/✘）。
  **G. 完整知识网络 ASCII 图**：所有关键概念/立场/裂缝及其关系。

**7. 会议纪要沉淀（默认开启）**：终局后，主持人指派库录入席判断本场是否产生**可复用结论**；
  若有，以 `input_type: conclusion` 经 `library-integrate` 创建结论记录并回写相关领域知识页。
  纯场次摘要不入库；触发重大变更时按 `library-integrate` 走待确认门禁，不自动覆盖。
  **若库录入席返回 `pending_confirmation`，主持人须在收尾向用户点明"有 N 项重大变更待你确认（见 `library/维护/待确认重大变更.md`）"，绝不替用户拍板。**
  若库内出现明显重复/碎片/失效链接累积，可另行触发 `library-maintain`。

**8. 无损存档（单文件自包含）**：生成时间戳（Windows PowerShell：`Get-Date -Format yyyyMMddTHHmmss`；其它环境：`date +%Y%m%dT%H%M%S`），
  把**全程原文**写入**单一文件** `discussions/{timestamp}--圆桌-{议题关键词}.md`，**不压缩、不删改任何发言**，并报告路径。
  必含且**原样内嵌**：事实底座**全本** + 各轮**每席发言原文** + 各轮综述与 **ASCII 图** + 终局 A–G 全文 + 终局核查全文。
  > **自包含 = 单文件可独立读全**：严禁把任何部分外移到别处再"引用"——
  > 既不得写"详见对话正文/见上文"，**也不得写"见某工作区/某外部文件"**（持久化到别处再引用同样不算自包含）。
  > **落盘位置固定**：本场若需中间草稿落盘（在子代理间传递大段原文/共享事实底座），只能放
  > `discussions/{timestamp}--圆桌-{议题关键词}.工作区/` 子目录；**严禁写到项目根目录或 `discussions/` 以外任何位置**。
  > **收尾清理**：把工作区内容全部并入上述单文件后，**删除该 `.工作区/` 子目录**，使 `discussions/` 下每场只留一个自包含存档。
  > 存档是原始全程留痕；知识库是蒸馏后的复用精华，二者并行、互不替代。

## 六、子代理调用模板

**事实席（开场建中立骨架）**：
```
subagent_type: generalPurpose, readonly: false（需联网）
任务：就议题「{议题}」联网建立《事实底座》中立骨架。只放客观共识、只给事实不给观点，
**不预判任何方法论该看什么数据**。输出：
1) 事实时间线（日期+事件）；2) 被各方引用的关键数据（数值+单位+日期+来源链接）；
3) 各方公开表态/立场（带来源）；4) 已知争议点；5) 证据硬度（硬/中/软），单一来源者标注。
要求：每条事实都带来源；区分"已确认/有争议/传闻"。
```

**事实席（按嘉宾数据需求补采）**：
```
subagent_type: generalPurpose, readonly: false（需联网）
任务：仅就以下嘉宾申报的【数据需求】定向补采，**不自行扩大范围**。逐条给出：
数值+单位+日期+来源链接+证据硬度；查不到的标"未能证实（含原因）"。并入现有底座，统一去重与来源把关。
【数据需求清单】…
```

**嘉宾席（每席并行）**：
```
subagent_type: generalPurpose, readonly: false（可联网核对；不要写改任何项目文件，不读写知识库；专有数据走【数据需求】申报、交事实席补采，不必自行扩大搜索）
任务：
1) 先用 Read 读取你的方法论：{该 lens 的 SKILL.md 路径——与本 host 同级，如 `<skills 目录>/<name>/SKILL.md`，由主持人按自身所在位置填入}，严格按其公理/框架/检查清单思考。
2) 阅读【本轮问题】【上一轮各席发言全文】【事实底座】。
3) 就本轮问题给出分析。【立场红线】你的立场/结论必须由你依据方法论与事实**自行推导**，
   本任务不会给你预设立场；首轮须声明【立场｜关键依据｜信心度与最大不确定源】。
   非首轮须【显式回应他席】至少一处（质疑/反驳/补充/修正），不许自说自话，并**重估本方立场**：
   较上轮是否变化、因何而变。
   【探讨非辩论】被新事实或对方论点说服而**改变立场或下调信心，是成果不是认输**；
   严禁为护住既有立场而选择性引用事实。
4) 数据须带来源。若需本方法论专有的事实/数据，用「【数据需求｜为何需要】」块**申报**（由事实席集中补采），不必自行扩大搜索；若你已掌握有据新事实，可用「【新增事实｜来源】」回传。
5) 结尾给「**简言之**：」一句话。方法论要锋利，但对事实与对方论点保持开放，不说正确的废话。
【本轮问题】…【上一轮发言】…【事实底座】…
```

**库录入席（材料/纪要，默认启用）**：
```
subagent_type: generalPurpose, readonly: false（需 Obsidian CLI 访问 library/）
任务：
1) 先用 Read 读取 `library-integrate/SKILL.md`（与本 host 同级），严格按其工作流与门禁执行；纯维护类需求改读 `library-maintain/SKILL.md`。
2) 把以下输入沉淀进个人知识库：
   - input_type: source | conclusion
   - 领域、主题、内容/结论、日期、来源（**尽量含 URL/出处**；确无链接则注明口径/出处与获取途径）、置信度与开放问题（如知）。
要求：用 Obsidian CLI（vault=everything-roundtable）先搜后写、整合优先、去重合并；

```

> 准确性优先的取舍：每轮**真的开子代理**（独立上下文）、统一事实底座、允许多轮深挖、终局核查、
> 引用留痕——以更多 token 换更少的事实漂移与确认偏误。

## 七、主持人行为准则（碰撞与纠错的精髓）

- **理性之锚**：不偏袒任何一席，也不偏袒任何结论。
- **不指派立场（红线）**：主持人只统一概念与事实、只**描述已经出现的分歧**，绝不替任何席预设立场、
  结论或"该站哪边"。立场是各席用各自方法论从事实里**推导/涌现**出来的，裂缝是被发现的，不是被规定的。
  若某席尚未给出明确立场，引导它"基于你的方法论，对本议题你的判断是什么"，而非告诉它判断。
- **挖深不铺广**：每轮只追**一条最深的裂缝**；先判定裂缝性质（事实/口径/假设），不同性质用不同方式破。
- **探讨而非辩论（求真而非求胜）**：共同求真、没有赢家；不为任何立场背书，让结论被事实逼出来；
  鼓励见骨追问但**对事开放**——欢迎任何一席因事实而改变立场。拒绝表面共识，
  也点破"假分歧/真共识"与"假共识/真分歧"。
- **追踪立场移动**：每轮记录各席立场/信心是否移动、因何而动；区分"被事实逼出的真趋同" vs
  "没触及根本的表面附和"。**既不强求一致，也不掩盖真共识；多轮后各持己见是完全可接受的结局。**
- **元认知**：综述要暴露讨论的**结构**（各方隐藏前提、推理链、被忽略的变量），不只复述。
- **事实纪律**：任何关键数字/因果，无来源即标"待核"；冲突的事实送事实席核查，不靠主持人拍板。
- **方法论忠诚**：确保每席真的在用其方法论的招式（先 Read 其 SKILL.md），而非泛泛而谈。

## 八、参会者发言格式

```
【{席位}】【行动标签】：……（含数据与来源）……
【立场｜信心】本轮：…（较上轮：持平/上调/下调，因…）
【数据需求｜为何需要】：（如有，交事实席补采）
【新增事实｜来源】：（如有，自带有据新事实）
**简言之**：一句话
```
行动标签：`陈述` / `质疑` / `反驳` / `补充` / `修正` / `更新立场` / `让步` / `存疑` / `综合`。
每段必须有"简言之"。**因事实或对方论点而改变立场、下调信心，用 `更新立场`/`让步` 标注——
这是被鼓励的探讨成果，不是失败。**

## 九、终局呈现规范（给用户看的"呈现层"，区别于无损存档）

- **倒金字塔**：永远**先结论、后过程**。顺序固定为 总论 → 对比速览表 → 趋同/分歧图 →（过程层）
  推理链 / 立场演化 / 开放问题 / 知识网络图。用户不往下读也已拿到全部结论。
- **排版纪律（反"大段文字"）**：能用**表 / 点列 / ASCII 图**就不写段落；确需文字时**单段 ≤3 行**；
  一句话能说清的不展开成一段。
- **对比优先**：多方**结论**用对比表并排；多方**思考过程**用点列骨架（依据→结论），不混成叙述。
- **信心可视化口径**：统一用五格条 + 百分比，如 `▮▮▮▮▯ 80%`（▮=已填，▯=空；就近取整到 10%）。
- **过程可折叠**：D–G 属"想看再读"；若用户只要结论，主持人可只呈现 A–C 并提示"要看分析过程可让我展开"。
- **存档不受影响**：本节只规范**对话里给用户看的内容**；步骤 8 的归档仍是**全程无损、单文件自包含原文**。

## 十、什么值得沉淀知识库（库录入席判定，配合 library-integrate）

只沉淀**未来跨场次可复用**的内容；判定与写法以 `library-integrate` 为准，本节给圆桌语境的取舍标尺：

- **值得（来源）**：支撑/挑战一个重要判断、提供可复用数据或验证路径、后续很可能再引用的精选来源。
  **来源尽量带 URL/出处**（机构+文件名+日期）；确无可链时注明"口径/出处"与获取途径，便于日后溯源复用。
- **值得（结论）**：可复用的机制/因果、决策规则、关键分歧及其根因、可证伪的开放问题、被新视角推翻或修正的认知。
- **不值得**：泛泛口号与风格模仿、逐字稿片段、无来源的传闻（除非标"待核来源+验证路径"）、
  无日期/来源的时效性事实、一次性私人偏好、别处已更清晰记录的重复结论、依赖内幕或非法获取的结论。

> 领域知识页表达当前最佳认知；结论记录保留当时判断并回链领域页。重大变更（核心结论反转、
> 置信度跨级、删除仍有依据内容、可靠来源冲突）一律走待确认门禁，不自动覆盖。

## 十一、每场自检清单

- [ ] 是否**动态选角**并由用户拍板，且阵容贴合议题、保留了视角多样性？
- [ ] 开场是否只建**中立骨架**，且后续**采集范围由嘉宾【数据需求】驱动**（非主持人/事实席自定）？关键数据带来源且标了证据硬度？
- [ ] 各席是否在**独立子代理**里分析、加载了各自方法论、每轮**显式回应了他席**？
- [ ] 主持人是否每轮锁定**一条最深裂缝**并判定其性质（事实/口径/假设）？
- [ ] ASCII 图是否呈现**结构**而非复述内容？
- [ ] 终局是否跑了**事实核查轮**，关键数字标了 ✔/⚠/✘？
- [ ] 存疑项是否进入**开放问题/风险**，而非被强行收敛成结论？
- [ ] 是否记录了各席**立场/信心演化**，并把"因事实改变立场"当作成果而非认输？
- [ ] 终局是否明确判定**趋同 or 各持己见**、讲清分歧根因，而非强凑共识？
- [ ] 终局是否**先给"一句话总论 + 对比表"再展开过程**（倒金字塔）？
- [ ] 是否按默认把达标**来源/结论**经库录入席沉淀进知识库（整合优先、去重、重大变更走门禁，且 `pending_confirmation` 向用户点明）？
- [ ] 全程原文是否**无损存档为单一自包含文件**（事实底座全本+各轮发言原文+ASCII+终局 A–G+核查全文均内嵌；无"见对话/见工作区"外移），落在 `discussions/`，且中间 `.工作区/` 已清理？
